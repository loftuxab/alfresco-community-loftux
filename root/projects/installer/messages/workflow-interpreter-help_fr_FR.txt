##
##  Meta commandes 
## 

ok> help

    Affiche cette aide. 

ok> r 

    Répète la dernière commande. 

ok> user [<nom d'utilisateur>] 

    Change l'utilisateur pour <nom d'utilisateur>.  Si <nom d'utilisateur> n'est
    pas renseigné, l'utilisateur affiché est l'utilisateur courant. 

ok> use

    Affiche le contexte du workflow courant.

## 
##  Commandes de Définition du Workflow
## 

ok> deploy <definitionClassPath> 

    Déploie la définition du workflow vers le serveur Alfresco. 

    <definitionClassPath> class path de la définition du workflow. 

ok> redeploy

    Redéploie la dernière définition de workflow.

ok> show definitions 

    Liste toutes les définitions de workflow déployés. 

ok> use definition [<workflowDefId>] 

    Change pour utiliser la définition de workflow identifiée par <workflowDefId>.
    Si <workflowDefId> n'est pas renseigné, la définition de workflow affichée est
    celle actuellement utilisée. 

## 
##  Commandes sur les variables 
## 

ok> var 

    Affiche toutes les variables définies. 

ok> var <varName>[*]=<varValue> 

    Définit ou modifie une variable. 

    <varName> nom de la variable
    [*] si spécifié, définit une collection 
    <varValue> valeur de la variable (séparer par une virgule pour spécifier une liste de valeurs) 

    exemples : 

    set bpm:assignee*=admin,fred 
    set wf:notifyMe=true 

ok> var <varName>[*] person <varValue> 

    Définit ou modifie une variable référençant un élément (cm:person). 

    <varName> nom de la variable
    [*] si spécifié, définit une collection 
    <varValue> valeur de la variable (séparer par une virgule pour spécifier une liste de valeurs) 

    exemple :
    
    set bpm:assignee* person admin,fred 

ok> var <varName>= 

    Efface une variable existante. 

    <varName> nom de la variable 

## 
##  Commandes de Workflow
## 
    
ok> start [<varName[=varValue>]]* 

    Démarre un nouveau workflow utilisant la définition de workflow actuellement
    sélectionnée. Les paramêtres de démarrage des tâches sont fournis par le
    couple nom/valeur ou utilisent les variables pré-définies. 
    
    exemple : 
    
    start bpm:assignee=david wf:predefined 
    
ok> show workflows 

    Affiche la liste des workflows actifs pour la définition de workflow sélectionnée.

ok> use workflow <workflowId>

    Utilise le workflow <workflowId> spécifié.

ok> show paths [<workflowId>] 

    Affiche le chemin du workflow pour le <workflowId> spécifié.  Si <workflowId> 
    n'est pas renseigné, le chemin affiché est celui du workflow actuellement démarré. 

ok> show transitions [<workflowId>] 

    Affiche toutes les transitions disponibles pour le <workflowId> spécifié. Si 
    <workflowId> n'est pas spécifié, les transitions affichées sont celles du
    workflow actuellement démarré.

ok> signal <pathId> [<transitionName>] 

    Signale la transition sur le <pathId> spécifié. Si <transitionName> n'est pas
    spécifié, la transition par défaut est utilisée. 

ok> desc workflow <workflowId> 

    Décrit le <workflowId> spécifié. 

ok> end workflow <workflowId> 

    Termine (cancel) le <workflowId> spécifié. 

## 
##  Commandes de tâches 
## 

ok> show my tasks 

    Affiche les tâches assignées à l'utilisateur actuellement sélectionné.

ok> show my completed 

    Affiche les tâches terminées par l'utilisateur actuellement sélectionné. 

ok> show tasks [<pathId>] 

    Affiche les tâches associées avec les <pathId> spécifiés. Si <pathId> n'est pas
    spécifié, les tâches affichées sont celles associées au workflow actuellement
    sélectionné.

ok> desc task <taskId> 

    Décrit la tâche identifiée par <taskId>. 

ok> update task <taskid> [<varName[=varValue>]]* 

    Met à jour l'état de la tâche <taskId>.  Les propriétés de la tâche sont fournies par le
    couple nom/valeur ou utilisent les variables pré-définies.

    <varName> nom de la variable 
    [*] si spécifié, définit une collection 
    <varValue> valeur de la variable (séparer par une virgule pour spécifier une liste de valeurs)
    
    exemple : 
    
    update task jbpm$122 bpm:assignee=fred wf:notifyMe=false 

ok> end task <taskId> [<transitionName>] 

    Termine une tâche identifiée par <taskId>.  Si <transitionName> n'est pas spécifié,
    la transition par défaut est utilisée.
